成り立ち（本音）

Why Noppo Auth?

Noppo Authは、Noppo Studioが開発している **NoppoBrowser** や **NoppoDT** （1対1のファイル転送サービス）といった複数のツール間で、「いちいちログインし直すのが面倒くさい」という、極めて個人的で切実な問題を解決するために作られました。

最初は単なる内部向けの「使い回しログイン」でしたが、自作ツールを公開する際にわざわざ独自の認証を組む手間に嫌気がさし、「どうせなら誰でも使えるように共通化してしまおう」という考えから今の形になっています。

美談はありません。ただ、私たちが欲しかった機能を、あなたもそのまま使えるようにしただけです。GitHubにコードを探しに行く必要もありません。ここにあるコードだけで、あなたのツールにもログイン機能を付けられます。

Benefits

Noppo Authを使うメリット

一度のログインでOK

NoppoStudio系サービスを始め、対応サイトならどこでも再入力不要でログイン状態を維持。

スマートな会員登録

メールアドレスや設定した表示名がスムーズに連携されるため、面倒なフォーム入力は不要です。

自分らしいIDで

共通プロフィール（display_name）により、どのサイトでも「いつもの自分」として即座に活動開始。

3 ユーザー情報の取得 

// 1. URLからトークンを抽出して保存する (ページロード時に実行)
function checkToken() {
    const token = new URLSearchParams(window.location.search).get('token');
    if (token) {
        localStorage.setItem('noppo_token', token);
        // URLを綺麗にする
        window.history.replaceState({}, document.title, window.location.origin + window.location.pathname);
    }
}

// 2. トークンを使ってユーザーデータを取得
async function getUser() {
    const token = localStorage.getItem('noppo_token');
    if (!token) return null;
    const res = await fetch(`${SUPABASE_URL}/auth/v1/user`, {
        headers: { "Authorization": `Bearer ${token}`, "apikey": SUPABASE_KEY }
    });
    return res.ok ? await res.json() : null;
}

利用規約・プライバシーポリシー

利用規約

1. サービスの概要

本サービスは、Noppo Studioが開発するアプリケーションおよび外部の開発者が公開するウェブサイトにおいて、共通のIDでログインを可能にするオープンソース認証基盤です。

2. 禁止事項

  • 他人のアカウント情報の不正使用。
  • サーバーへの過度な負荷をかける行為。
  • 運営を妨害する行為。

3. 免責事項

当組織は、本サービスに起因してユーザーに生じたあらゆる損害について、一切の責任を負いません。本サービスは現状のまま提供されます。

プライバシーポリシー

1. 収集する情報

本サービスでは、メールアドレス、表示名（display_name）、認証トークン、ログイン日時を収集・保持します。

2. 情報の共有（外部サイトへの提供）

ユーザーが本サービスを利用して外部サイトにログインした場合、メールアドレス、表示名、一意のユーザーIDが提供されます。ログイン時の「許可」により同意したものとみなされます。

3. セキュリティ

本サービスはSupabaseの強固なインフラを使用しており, パスワードは暗号化され、当組織が直接閲覧することはありません。
